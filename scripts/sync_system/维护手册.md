## 1. 系统概述

### 1.1 功能介绍

自动化定时任务系统用于定期同步原项目 (lingfengQAQ/webnovel-writer) 的文档更新，主要功能包括:

- **定时同步机制**: 支持每日/每周/每小时同步，增量获取变更
- **版本号跟踪**: 确保与原项目主版本号精确匹配
- **兼容性测试**: 同步后自动触发测试套件
- **依赖管理**: 分类标注必选/可选依赖
- **异常处理**: 自动重试机制和通知系统

### 1.2 系统架构

```
sync_system/
├── sync_scheduler.py        # 同步调度器
├── compatibility_tester.py  # 兼容性测试套件
├── dependency_manager.py    # 依赖管理器
├── notification.py          # 通知系统
├── sync_system.py           # 主集成模块
├── config/
│   ├── sync_config.ini      # 主配置文件
│   └── windows_task_scheduler.ps1  # Windows 计划任务脚本
├── tests/
│   └── test_sync_system.py  # 单元测试
└── logs/                     # 日志目录
```

## 2. 安装与配置

### 2.1 环境要求

- Python 3.8+
- requests 库
- Git (用于版本控制)
- Windows Task Scheduler 或 cron (用于定时任务)

### 2.2 配置文件

编辑 `config/sync_config.ini`:

```ini
[sync_source]
repo_url = https://github.com/lingfengQAQ/webnovel-writer
branch = master
docs_paths = docs, webnovel-writer/docs

[schedule]
frequency = daily
daily_time = 03:00

[notifications]
enabled = true
channels = console, file
```

## 3. 使用指南

### 3.1 基本命令

```bash
# 进入同步系统目录
cd scripts/sync_system

# 查看状态
python sync_system.py status

# 执行单次同步
python sync_system.py sync

# 强制同步 (忽略时间检查)
python sync_system.py sync --force

# 运行完整周期 (同步 + 测试 + 依赖检查)
python sync_system.py run

# 运行测试
python sync_system.py test

# 检查依赖
python sync_system.py dep

# 生成依赖清单
python sync_system.py dep --manifest

# 启动调度器
python sync_system.py start
```

### 3.2 运行测试

```bash
# 运行所有测试
python -m pytest tests/

# 运行同步系统测试
python tests/test_sync_system.py

# 测试特定模块
python -m pytest tests/test_sync_system.py::TestSyncScheduler -v
```

## 4. 定时任务配置

### 4.1 Windows 任务计划程序

以管理员身份打开 PowerShell:

```powershell
cd E:\小说\.trae\skills\webnovel-writer\scripts\sync_system\config

# 创建每日同步任务
.\windows_task_scheduler.ps1
```

### 4.2 手动创建任务

```powershell
schtasks /create /tn "WebnovelWriterSync" /tr "python.exe E:\小说\.trae\skills\webnovel-writer\scripts\sync_system\sync_system.py run" /sc daily /st 03:00 /f
```

### 4.3 查看和删除任务

```powershell
# 查看任务
schtasks /query /tn "WebnovelWriterSync"

# 删除任务
schtasks /delete /tn "WebnovelWriterSync" /f
```

## 5. 常见问题处理

### 5.1 同步失败

**问题**: `sync_status = failed`

**可能原因**:
1. 网络连接问题
2. GitHub API 限流
3. 认证凭据过期

**解决方案**:
```bash
# 1. 检查网络
ping github.com

# 2. 查看详细日志
type logs\sync_2026-01-01.log

# 3. 手动重试
python sync_system.py sync --force
```

### 5.2 测试失败

**问题**: `test_status = failed`

**可能原因**:
1. 文档格式不兼容
2. 链接失效
3. 代码语法错误

**解决方案**:
```bash
# 查看测试报告
type tests\reports\test_report_*.json

# 检查具体失败项
python sync_system.py test
```

### 5.3 依赖问题

**问题**: `missing > 0`

**解决方案**:
```bash
# 查看缺失依赖
python sync_system.py dep

# 安装必选依赖
pip install aiohttp filelock pydantic requests

# 生成完整依赖清单
python sync_system.py dep --manifest
```

### 5.4 通知发送失败

**问题**: Webhook/Email 通知失败

**解决方案**:
1. 检查配置文件中的 Webhook URL 和 Email 设置
2. 确认网络连接
3. 查看通知日志: `logs/notification_*.log`

### 5.5 版本不匹配

**问题**: `version_mismatch`

**可能原因**:
1. 原项目已更新版本号
2. 同步缓存过期

**解决方案**:
```bash
# 强制重新同步
python sync_system.py sync --force

# 清理缓存
Remove-Item -Recurse .sync_cache -Force
Remove-Item -Recurse .sync_temp -Force
```

## 6. 维护指南

### 6.1 日志管理

日志文件位置: `scripts/sync_system/logs/`

```powershell
# 清理 30 天前的日志
Get-ChildItem logs\*.log | Where-Object { $_.LastWriteTime -lt (Get-Date).AddDays(-30) } | Remove-Item
```

### 6.2 缓存清理

```powershell
# 清理同步缓存
Remove-Item -Recurse .sync_cache -Force -ErrorAction SilentlyContinue
Remove-Item -Recurse .sync_temp -Force -ErrorAction SilentlyContinue
```

### 6.3 状态重置

```powershell
# 重置同步状态 (慎用)
Remove-Item -Recurse .sync_status -Force -ErrorAction SilentlyContinue
```

### 6.4 升级系统

```bash
# 1. 备份配置
Copy-Item config/sync_config.ini config/sync_config.ini.bak

# 2. 更新代码
git pull

# 3. 恢复配置
Copy-Item config/sync_config.ini.bak config/sync_config.ini

# 4. 运行测试
python sync_system.py test
```

## 7. 附录

### 7.1 依赖分类清单

#### 必选依赖 (Required)

| 包名 | 版本 | 用途 | 使用场景 |
|------|------|------|----------|
| aiohttp | >=3.8.0 | 异步 HTTP 客户端 | RAG API 调用 |
| filelock | >=3.0.0 | 文件锁 | 多进程并发控制 |
| pydantic | >=2.0.0 | 数据验证 | Story System 模型 |
| requests | - | HTTP 请求 | GitHub API 调用 |

#### 可选依赖 (Optional)

| 包名 | 用途 | 使用场景 | 限制条件 |
|------|------|----------|----------|
| chromadb | 向量数据库 | RAG 向量存储 | 需要 4GB+ 内存 |
| flask | Web 框架 | Dashboard | 可选，CLI 不需要 |
| flask-cors | CORS 支持 | Dashboard 跨域 | 仅配合 Flask |

#### 开发依赖 (Development)

| 包名 | 用途 |
|------|------|
| pytest | 测试框架 |
| pytest-cov | 覆盖率报告 |

#### 云服务依赖 (Cloud Services)

| 包名 | 用途 | 条件 |
|------|------|------|
| openai | GPT 模型 | 需要 API Key |
| jina | Rerank API | 需要 API Key |

### 7.2 配置文件完整示例

```ini
# Webnovel Writer Sync System Configuration

[sync_source]
repo_url = https://github.com/lingfengQAQ/webnovel-writer
branch = master
docs_paths = docs, webnovel-writer/docs, webnovel-writer/scripts
ignore_patterns = *.tmp, *.bak, __pycache__, .git

[schedule]
frequency = daily
weekly_day = 0
daily_time = 03:00
incremental_window_hours = 24

[version_tracking]
version_file = .sync_status/version.json
strict_version_match = true
check_interval = 60
auto_rollback = true

[testing]
trigger = on_change
timeout_seconds = 300
parallel_workers = 4
generate_coverage = true
max_test_reports = 10

[testing.scope]
check_content_accuracy = true
check_format_compatibility = true
check_link_validity = true
check_code_syntax = true
check_image_refs = true

[dependencies]
manifest_path = requirements.txt
optional_manifest_path = requirements-optional.txt
lock_versions = true
strict_mode = false

[notifications]
enabled = true
channels = console, file

[error_handling]
max_retries = 3
retry_interval = 300
exponential_backoff = true
auto_recovery = true

[logging]
log_dir = scripts/sync_system/logs
level = INFO
format = %(asctime)s - %(name)s - %(levelname)s - %(message)s
max_file_size = 10
backup_count = 30
console_output = true
```
